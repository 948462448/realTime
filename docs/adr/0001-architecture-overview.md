# ADR-0001: Android 实时视频字幕翻译 App

## 状态

已采纳

## 背景

需要一款 Android App，能够对无字幕视频（直播、本地视频、在线视频）实时添加翻译字幕。核心链路为：音频 → 语音识别(ASR) → 翻译 → 字幕渲染。

## 关键约束

- **全离线运行**（可选），ASR 和翻译均在手机端完成
- **实时性**，字幕延迟在可接受范围内
- **多语言**，覆盖中英文及常见语言

## 技术选型

### 语音识别: Whisper.cpp

- 开源成熟，Android 端通过 JNI 集成
- 支持 99 种语言，准确率高
- 有现成的 Android 示例和构建脚本
- 使用 `tiny` 或 `base` 模型平衡速度和准确率

### 翻译引擎: Tencent Hy-MT2-1.8B

- 专为翻译设计的 1.8B 模型，非通用 LLM，翻译质量更好
- 1.25-bit 量化后仅 440MB，可手机部署
- 支持 33 种语言互译
- GGUF 格式，通过 llama.cpp 推理

### 视频播放: ExoPlayer (Media3)

- Android 官方媒体库，支持本地/在线视频
- 可自定义渲染器，方便提取音频流
- 支持字幕轨道叠加

### 字幕渲染: Compose Overlay

- Jetpack Compose 层叠在播放器之上
- 实时显示识别+翻译结果
- 可自定义样式（字体、颜色、位置）

## 工作流程

```
1. 用户打开视频（本地/URL/直播流）
2. ExoPlayer 开始播放
3. AudioPipe 持续抓取解码后的 PCM 音频
4. 音频分片(～30s)送入 Whisper.cpp 做 ASR
5. ASR 结果送入 Hy-MT2 翻译
6. 译文回传到 UI 层叠加显示
7. 字幕随播放进度滚动
```

## 渐进式实现计划

### Phase 1: 基础播放 + 展示
- Android 项目脚手架（Kotlin + Compose）
- ExoPlayer 集成，支持本地/URL 播放
- 字幕 UI 组件（手动输入字幕验证渲染）

### Phase 2: ASR 集成
- 编译 Whisper.cpp for Android (arm64)
- JNI 封装，音频采集管道
- 实时语音识别 + 原文显示

### Phase 3: 翻译集成
- 下载 Hy-MT2-1.8B GGUF 模型
- 编译 llama.cpp for Android
- ASR 结果 → 翻译 → 字幕显示 全链路

### Phase 4: 体验优化
- 延迟优化（流式 ASR、增量翻译）
- 多语言切换
- 直播流支持
- UI 美化
